# 修复测试脚本中的类型错误方案

## 1. 问题分析

测试脚本在运行时出现错误：`TypeError: Could not convert ['010203'] to numeric`，这是因为：

- 对于 BTAD 数据集，class_name 是 "01"、"02"、"03" 这样的字符串
- 当调用 `df.mean()` 时，pandas 尝试对所有列计算平均值，包括 "class name" 列
- 但是 "class name" 列包含字符串值，无法转换为数值类型进行平均计算

## 2. 解决方案

修改 test.py 文件，只对数值列计算平均值，跳过 "class name" 列。

## 3. 实现步骤

1. **修改 test.py 文件**：在计算平均值之前，从 DataFrame 中选择只包含数值的列
2. **确保平均值计算正确**：只对 "pixel AUC"、"pixel AP"、"image AUC"、"image AP" 这些数值列计算平均值
3. **保持其他功能不变**：确保修改后的代码仍然能够正确生成和保存测试结果

## 4. 预期效果

修改后的代码能够正确计算 DataFrame 的平均值，不会尝试对字符串列进行数值计算，从而避免类型错误。测试脚本可以正常运行，为所有数据集生成测试结果。
